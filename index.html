Section 02 · Prime
P

Prime — load the agent's context

  • Does: builds a tailored primer — stack, style, constraints
  • Prevents: the agent guessing your architecture
  • Skill: /prime → a context block read before any code

RULE — Never start a session without priming.

Section 02 · Prime

What a primer looks like

primer.md
Stack:        Next.js 14 · PostgreSQL · Tailwind
Style:        TypeScript strict · no `any` · named exports
Constraints:  no new dependencies without approval
Threat model: public-facing · no auth bypass
Goal:         add the invoice export endpoint

30 seconds of work that saves 2 hours of correction.

U is not a spec review.
U is a conversation with a purpose.

Section 02 · Understand
U

Understand — clarify before code

  • Does: asks targeted questions, one at a time
  • Prevents: building the wrong thing perfectly
  • Skill: /understand

If you can't answer the question, the scope isn't ready.

Section 02 · War Story

“I built the entire user settings page.

  • 4-hour rework — because U was skipped
  • The one question that catches it: “Is this per-user or per-account?”

R is the phase most developers skip.
It's also the one that ships quality.

Section 02 · Refine
R

Refine — senior reviewer pass

  • Does: systematic audit of AI output before commit
  • Prevents: bugs that “look correct” but aren't
  • Skill: /refine — audit → fix list → re-audit

Not a vibe check. A real review.

Section 02 · Refine

What refine catches
that a first read misses

  • Edge cases the agent never asked about
  • Performance assumptions baked into the approach
  • Security holes that read like valid code
  • Missing error handling at system boundaries

E is where the work happens.
Everything before it determines the quality.

Section 02 · Execute
E

Execute — delegate with precision

  • Does: hands a scoped task to a coding agent, tracks the run
  • Prevents: open-ended delegation that drifts
  • Contract: clear input → tracked output → verified result
  • Plugs into: Claude Code · Cursor · Codex · Aider · Windsurf

Section 05 · The Traps

The 5 failure patterns

  • Skip Prime → agent hallucinates your stack
  • Skip Understand → you build the right thing wrong
  • One-pass Refine → bugs ship a 2nd read would catch
  • Open-ended Execute → agent drifts, scope inflates
  • No doctrine file → every session starts from scratch

“Discipline is what makes vibe coding
scale beyond one developer.”

Section 05 · The Fixes

Five fixes — one per failure

  • Prime always — even for “quick fixes”
  • Understand — one unclear assumption = one more loop
  • Refine — audit → fix list → re-audit. Not one pass.
  • Execute — scoped input · tracked output · verified result
  • Doctrine — every rule you earn goes in the file tonight
